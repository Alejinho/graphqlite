Use PHP Annotations to declare your GraphQL API

Create a complete GraphQL API by simply annotating your PHP classes.

You focus on your PHP code.

GraphQLite comes with an advanced type-mapper that automatically translate your PHP types into GraphQL types.

/**
 * @Type()
 */
class Product
{
    /**
     * @Field()
     */
    public function getName(): string
    {
        return $this->name;
    }

Annotation based

Almost no code needed to switch your API to GraphQL!

Framework agnostic

Can be installed in any PHP project

... but Symfony bundle available

Native integration with Symfony provided

Comes with batteries included

  • Queries
  • Mutations
  • Mapping of arrays / iterators
  • Support for input types
  • Extendable types
  • Plugs into your framework security module
  • Map PHP inheritance to GraphQL interfaces automatically
  • Union types
  • GraphQL file uploads
  • Native DateTime type

Get started!

  1. Declare a query in your controller

Annotate the query with the @Query annotation

class ProductController
{
    /**
     * @Query
     */
    public function product(string $id): Product
    {
        // Some code that looks for a product and returns it.
    }
}

/**
 * @Type()
 */
class Product
{
    /**
     * @Field()
     */
    public function getName(): string
    {
        return $this->name;
    }
    // ...
}

  1. Annotate your types

Annotate the Product class to declare what fields are exposed to the GraphQL API

  1. Query and enjoy

You're good to go! Query and enjoy!

{
  product(id: 42) {
    name
  }
}
